# CLAUDE.md ‚Äî CS-6180 Project Gallery

## Project Overview

Static project gallery for **CS-6180** deployed on **AWS Amplify**.
No backend, no framework ‚Äî vanilla HTML/CSS/JS.
URL: each team gets `/teams/{slug}/index.html` auto-loaded by `viewer.js`.

---

## Architecture

| Layer | Detail |
|---|---|
| Discovery | `teams/manifest.json` ‚Äî auto-generated by `scripts/build.js` on every push |
| Rendering | `viewer.js` builds the full viewer page at runtime from `project.json` |
| Markdown | `project-loader.js` renders README.md client-side via marked.js + DOMPurify (CDN) |
| Deployment | AWS Amplify, `amplify.yml` runs `git lfs pull` then `node scripts/build.js` |
| LFS | Posters (PNG) and videos (MP4) are stored in Git LFS |

---

## Key Files

```
amplify.yml              ‚Äî preBuild: install git-lfs binary, lfs pull, build.js
scripts/build.js         ‚Äî scans teams/, writes manifest.json (skips _-prefixed dirs)
scripts/new-team.js      ‚Äî CLI to scaffold a new team folder
js/viewer.js             ‚Äî builds viewer HTML, handles Poster/Video tabs, PiP, swipe
js/gallery.js            ‚Äî renders the card grid with search/filter
js/project-loader.js     ‚Äî fetches README.md, rewrites relative paths, wraps iframes
styles/viewer.css        ‚Äî full-page viewer layout, tabs, PiP, mobile tweaks
teams/_template/         ‚Äî copy this to add a new team
teams/manifest.json      ‚Äî auto-committed by GitHub Actions on every push to main
```

## TA Workflow ‚Äî Adding a Team

```bash
node scripts/new-team.js team-01 "Team Name"
# ‚Üí creates teams/team-01/ with project.json + README.md
# ‚Üí regenerates manifest.json
```

Or manually:
1. Copy `teams/_template/` ‚Üí `teams/{slug}/`
2. Edit `project.json` (name, title, posterUrl, videoUrl)
3. Add `poster.png` and `video.mp4` (tracked by Git LFS)
4. Run `node scripts/build.js` to update `manifest.json`
5. Commit + push ‚Äî Amplify auto-deploys

## project.json Schema

```json
{
  "name": "Team 01",
  "title": "Project title shown in header",
  "posterUrl": "poster.png",
  "videoUrl":  "video.mp4"
}
```

- `posterUrl` ‚Äî prefer `.png` (converted from PDF); `.pdf` works as desktop fallback only
- `videoUrl`  ‚Äî `.mp4` preferred; iframe embeds (YouTube) also work

## Viewer UI ‚Äî How It Works

`viewer.js` injects a full-page viewer with:

- **Header**: ‚Üê Gallery back link | NU logo | Team title | **POSTER / VIDEO** tabs
- **Body**: full-panel Poster or Video, toggled by tabs or swipe (mobile)
- **PiP** (picture-in-picture): draggable, resizable overlay showing the secondary content when both poster + video exist

### Modes
| Mode | Main panel | PiP |
|---|---|---|
| `mode-default` / `mode-poster` | Poster | Video |
| `mode-video` | Video | (hidden if no poster) |
| `mode-swapped` | Video | Poster |

### Controls
- **Tabs** (POSTER / VIDEO) ‚Äî switch mode
- **Swipe left/right** (mobile) ‚Äî switch mode
- **PiP dragbar** ‚Äî drag to reposition
- **‚§¢ maximize** ‚Äî expand PiP
- **‚áÑ swap** ‚Äî swap main ‚Üî PiP content
- **Resize handle** (bottom-right corner of PiP) ‚Äî resize PiP
- Keyboard: `P` = poster, `V` = video, `Space` = play/pause

## Mobile / WebView Notes

The app is embedded in **Metasteps** (Unity-based metaverse) via a WebView on mobile.

### PDF problem
PDFs do **not** open in mobile WebViews ‚Äî the WebView has no PDF plugin.
**Fix**: convert all posters from `.pdf` ‚Üí `.png` using `convert-posters.py` (ImageMagick).

```bash
python convert-posters.py   # converts poster.pdf ‚Üí poster.png in each team folder
```

`viewer.js` auto-detects extension:
- `.png` / `.jpg` / `.webp` ‚Üí `<img>` tag (works everywhere)
- `.pdf` ‚Üí `<embed>` tag (desktop only, blocked in WebView)

### Poster display on mobile
`styles/viewer.css` ‚Äî `.viewer-panel--poster` + `img.viewer-pdf`:
- `object-fit: contain` + `object-position: top center` ‚Äî poster fills panel maximally, anchored to top (no centering black bars)
- `background: #fff` on both panel and img ‚Äî eliminates black gaps below poster
- `overflow: hidden` ‚Äî no scroll into empty space

## Principles

- **Zero npm dependencies at runtime** ‚Äî CDN only for marked.js + DOMPurify
- **No build step for source** ‚Äî `build.js` only generates `manifest.json`
- **Push = deploy** ‚Äî GitHub Actions auto-commits manifest; Amplify auto-deploys on push to `main`
- **PNG over PDF** ‚Äî always convert before uploading; PDF is desktop-only fallback
