# CS-6180 Project Gallery

A static website for showcasing student final projects in **CS-6180 (Generative AI)** at Northeastern University. Deployed on **AWS Amplify**. No backend, no build framework — vanilla HTML/CSS/JS.

---

## System Architecture

This gallery is one component of a larger presentation system that spans a static website and a Unity-based virtual environment (Metasteps).

```
┌─────────────────────────────────────────────────────────────┐
│                     Static Gallery Website                  │
│  index.html ──→ teams/manifest.json ──→ gallery cards       │
│  teams/{slug}/  ──→ viewer.js  ──→ Poster / Video viewer    │
│                           └──→ qa.js  ──→ Supabase Q&A      │
└───────────────────────┬─────────────────────────────────────┘
                        │  unique URL per team
                        ▼
┌─────────────────────────────────────────────────────────────┐
│              Metasteps (Unity Metaverse)                    │
│                                                             │
│   Wall 01 ──→ WebView(https://<domain>/teams/team-01/)      │
│   Wall 02 ──→ WebView(https://<domain>/teams/team-02/)      │
│   …                                                         │
│   Wall 32 ──→ WebView(https://<domain>/teams/team-32/)      │
└─────────────────────────────────────────────────────────────┘
```

| Layer | Technology |
|-------|-----------|
| Hosting | AWS Amplify (static) |
| Discovery | `teams/manifest.json` — auto-generated by `scripts/build.js` |
| Viewer | `viewer.js` — full-page poster/video viewer with PiP |
| Markdown rendering | `project-loader.js` via marked.js + DOMPurify (CDN) |
| Q&A | Supabase (Postgres) — `js/qa.js` |
| In-world display | Metasteps WebView element, one wall per team |
| Assets | Git LFS — posters (PNG) and videos (MP4) |

## Metasteps Integration

### Overview

[Metasteps](https://metasteps.com) is a Unity-based virtual metaverse platform. The gallery is embedded inside a Metasteps virtual environment as a series of **WebView walls** — one wall per student team.

Each wall is an interactive in-world element that renders a web browser view pointed at a specific team's gallery URL. Visitors walk up to a wall in the virtual space and interact with the project (view poster, watch video, ask questions) directly within the metaverse.

### How it works

1. **Each team gets a unique URL** on the deployed gallery:
   ```
   https://<amplify-domain>/teams/team-01/
   https://<amplify-domain>/teams/team-02/
   …
   https://<amplify-domain>/teams/team-32/
   ```

2. **In Metasteps**, a dedicated virtual room or hall is set up with 32 wall objects. Each wall has a **WebView component** whose URL is set to the corresponding team's gallery URL.

3. **The viewer page** (`viewer.js`) auto-loads from `project.json` — no configuration needed on the Metasteps side beyond the URL.

4. **Mobile / WebView considerations** — the Metasteps WebView runs on mobile and does not have a PDF plugin. All posters must be PNG (not PDF). See [Poster format](#poster-format) below.

### Metasteps setup (per wall)

In the Metasteps editor / Unity scene:
- Add a **WebView interactive element** to the wall mesh
- Set the WebView URL to `https://<amplify-domain>/teams/team-XX/`
- No additional scripting is needed — the gallery page handles all UI

The main Metasteps scene declaration lives outside this repository (in the Metasteps project files). This repository only provides the URLs that the WebView elements point to.

## Supabase Q&A Integration

Each project viewer includes a **Q&A / Discussion panel** (the floating "Q&A" button) powered by [Supabase](https://supabase.com).

### Architecture

- `js/qa.js` — client-side JS loaded dynamically via CDN (`@supabase/supabase-js@2`)
- Supabase table: `questions` — stores threaded discussions per `team_slug`
- Anonymous key is used (read/write for any visitor — no auth required)

### Schema

```sql
create table questions (
  id          uuid primary key default gen_random_uuid(),
  team_slug   text not null,
  author_name text not null,
  email       text,
  body        text not null,
  parent_id   uuid references questions(id),
  depth       int  default 0,
  created_at  timestamptz default now()
);
```

### How it's wired

`viewer.js` loads the Supabase CDN script and then `qa.js` on demand when the viewer renders:

```js
loadScript('https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2/...', () =>
  loadScript('../../js/qa.js', () => window.initQA?.(_teamSlug, team.name))
);
```

`initQA(slug, teamName)` binds the FAB button and panel to load/post threads scoped to that team's slug.

### Configuration

The Supabase URL and anon key are stored at the top of `js/qa.js`:
```js
const SUPABASE_URL = 'https://<your-project>.supabase.co';
const SUPABASE_KEY = '<anon-key>';
```

To use a different Supabase project, replace these two values and run the schema migration above.

## Gallery Website

### Key Files

```
amplify.yml              — preBuild: install git-lfs, lfs pull, build.js
scripts/build.js         — scans teams/, writes manifest.json
scripts/new-team.js      — CLI to scaffold a new team folder
js/viewer.js             — full-page viewer (poster/video tabs, PiP, swipe)
js/gallery.js            — gallery card grid with search/filter
js/project-loader.js     — fetches README.md, rewrites paths, wraps iframes
js/qa.js                 — Supabase Q&A threads
styles/viewer.css        — viewer layout, tabs, PiP, mobile
teams/_template/         — copy to add a new team
teams/manifest.json      — auto-generated on every push
```

### For TAs: Adding a Team

**Option A — Script (recommended)**

```bash
node scripts/new-team.js team-01 "Team Name"
# → creates teams/team-01/ with project.json + README.md
# → regenerates manifest.json
```

**Option B — Manual**

1. Copy `teams/_template/` → `teams/{slug}/`
2. Edit `project.json` (name, title, posterUrl, videoUrl)
3. Add `poster.png` and `video.mp4` (tracked by Git LFS)
4. Run `node scripts/build.js` to update `manifest.json`
5. Commit + push — Amplify auto-deploys

### project.json Schema

```json
{
  "name":        "Team 01",
  "title":       "Project title shown in header",
  "members":     ["Student A", "Student B"],
  "description": "One-line summary for the gallery card.",
  "thumbnail":   "assets/thumbnail.png",
  "poster":      "assets/poster.png",
  "video":       "assets/video.mp4",
  "tags":        ["generative-ai", "rag"]
}
```

### Poster Format

Posters **must be PNG** (or JPG/WebP). PDF posters work on desktop browsers only and are silently blocked in the Metasteps WebView.

```bash
python convert-posters.py   # batch-converts poster.pdf → poster.png
```

`viewer.js` auto-detects the extension:
- `.png` / `.jpg` / `.webp` → `<img>` tag (works everywhere)
- `.pdf` → `<embed>` tag (desktop only, blocked in WebView)

### Viewer Modes

| Mode | Main panel | PiP overlay |
|------|-----------|-------------|
| `mode-poster` (default) | Poster | Video |
| `mode-video` | Video | — |
| `mode-swapped` | Video | Poster |

Controls: **POSTER / VIDEO** tabs, swipe left/right (mobile), `P` / `V` keyboard shortcuts, drag PiP, ⤢ maximize, ⇄ swap.

## Deployment

### AWS Amplify

Push to `main` → Amplify auto-deploys:
1. `amplify.yml` installs git-lfs binary, runs `git lfs pull`
2. Runs `node scripts/build.js` to regenerate `manifest.json`
3. Deploys all static files

### GitHub Actions

A workflow (`.github/workflows/build-manifest.yml`) auto-commits the updated `manifest.json` on every push so it stays in sync.

### Preview locally

```bash
npx serve .
# open http://localhost:3000
```

## Adding a New Course / Cohort

To reuse this gallery for a different semester or course:

1. Update course info in `index.html`, `staff.html`, and `northeastern.html`
2. Update `teams/` — run `node scripts/new-team.js` for each team
3. Upload poster PNGs and video MP4s via Git LFS
4. In Metasteps: update each wall's WebView URL to the new team's URL
5. For Q&A: create a new Supabase project, run the schema, update `js/qa.js`

---

## Principles

- **Zero npm at runtime** — CDN only (marked.js, DOMPurify, Supabase JS)
- **No build step for source** — `build.js` only generates `manifest.json`
- **Push = deploy** — GitHub Actions auto-commits manifest; Amplify auto-deploys
- **PNG over PDF** — always convert before uploading
