## System Architecture

This gallery is part of a larger presentation system combining a static website, a Supabase database, and a Unity-based virtual environment (Metasteps).

```
┌─────────────────────────────────────────────────────────────────┐
│                      Static Gallery Website                     │
│                         (AWS Amplify)                           │
│                                                                 │
│  index.html ──→ fetch(teams/manifest.json) ──→ gallery cards   │
│                                                                 │
│  teams/{slug}/  ──→  viewer.js  ──→  Poster / Video viewer     │
│                             └──→  qa.js  ──→  Supabase Q&A     │
└───────────────────────┬─────────────────────────────────────────┘
                        │  each team has a unique URL
                        ▼
┌─────────────────────────────────────────────────────────────────┐
│               Metasteps (Unity Metaverse)                       │
│                                                                 │
│   Wall 01 ──→ WebView → https://<domain>/teams/team-01/        │
│   Wall 02 ──→ WebView → https://<domain>/teams/team-02/        │
│   …                                                             │
│   Wall 32 ──→ WebView → https://<domain>/teams/team-32/        │
└─────────────────────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────────┐
│                    Supabase (Postgres)                          │
│                                                                 │
│   questions table ←→ qa.js ←→ viewer Q&A panel                 │
│   (threaded per team_slug)                                      │
└─────────────────────────────────────────────────────────────────┘
```

| Layer | Detail |
|-------|--------|
| Hosting | AWS Amplify — static, no server |
| Discovery | `teams/manifest.json` — auto-generated by `scripts/build.js` |
| Viewer | `js/viewer.js` — poster/video viewer with PiP, keyboard, touch |
| Markdown | `js/project-loader.js` — renders `README.md` via marked.js + DOMPurify (CDN) |
| Q&A | Supabase Postgres — `js/qa.js`, loaded via CDN |
| In-world | Metasteps WebView element per team wall |
| Assets | Git LFS — posters (PNG) and videos (MP4) |

## Metasteps Integration

### Overview

[Metasteps](https://metasteps.com) is a Unity-based virtual metaverse platform. The gallery is embedded inside a Metasteps virtual environment as a series of **interactive WebView walls** — one wall per student team.

Each wall is an in-world object with a WebView component that opens a web browser pointed at a team's gallery URL. Visitors walk up to a wall in the virtual space and can view the poster, watch the video, and post Q&A questions — entirely within the metaverse.

### How it works

1. **Each team gets a unique URL** when deployed:
   ```
   https://<amplify-domain>/teams/team-01/
   https://<amplify-domain>/teams/team-02/
   …
   https://<amplify-domain>/teams/team-32/
   ```

2. **In the Metasteps environment**, a virtual room or hallway contains 32 wall objects. Each wall has a **WebView component** whose source URL is set to the corresponding team's gallery URL above.

3. **No extra Metasteps code is needed per team** — the viewer page at the URL handles everything: loading `project.json`, rendering the poster/video, and wiring up Q&A.

4. **The Metasteps scene declarations** (Unity project files, wall layout, WebView binding) live outside this repository. This repo provides only the URLs that the WebView components point to.

### WebView mobile caveats

The Metasteps WebView runs on mobile and has no PDF plugin. All posters **must be PNG** (not PDF) — PDFs load silently as blank on mobile WebViews. See [Asset Guidelines](#asset-guidelines).

## Supabase Q&A Integration

Every project viewer has a floating **Q&A button** (bottom-right corner). Clicking it opens a threaded discussion panel backed by Supabase.

### How it is connected

`js/viewer.js` dynamically loads the Supabase CDN client and then `js/qa.js` after the viewer renders:

```js
loadScript(
  'https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2/dist/umd/supabase.min.js',
  () => loadScript('../../js/qa.js', () => window.initQA?.(_teamSlug, team.name))
);
```

`initQA(slug, teamName)` in `qa.js` binds the FAB and panel to load and post threads scoped to that team's slug (e.g. `"team-01"`).

### Supabase project configuration

The Supabase URL and anon key are at the top of `js/qa.js`:

```js
const SUPABASE_URL = 'https://<your-project>.supabase.co';
const SUPABASE_KEY = '<anon-key>';
```

The anon key is safe to expose in client-side code — it is read-only and scoped to the `questions` table by Row Level Security.

### Database schema

```sql
create table questions (
  id          uuid        primary key default gen_random_uuid(),
  team_slug   text        not null,
  author_name text        not null,
  email       text,
  body        text        not null,
  parent_id   uuid        references questions(id),
  depth       int         default 0,
  created_at  timestamptz default now()
);

-- Allow anyone to read and insert; no updates or deletes
create policy "public read"   on questions for select using (true);
create policy "public insert" on questions for insert with check (true);
alter table questions enable row level security;
```

### Q&A features

- **Threaded replies** — any post can be replied to, replies are visually indented (capped at depth 6)
- **Relative timestamps** — "just now", "5m ago", "2h ago", "3d ago"
- **No authentication required** — visitors enter a name (and optional email) per post
- **Draggable FAB** — the Q&A button can be dragged anywhere on the viewer page
- **Lazy load** — threads are fetched only when the panel is first opened

## Gallery Website

### Key files

```
index.html               — gallery landing page
staff.html               — course staff page
northeastern.html        — about / course info page
feedback.html            — anonymous feedback form (localStorage only)

js/gallery.js            — fetches manifest, renders card grid, handles search
js/viewer.js             — full-page project viewer (poster, video, PiP, Q&A)
js/project-loader.js     — fetches README.md, rewrites relative paths, wraps iframes
js/qa.js                 — Supabase Q&A threads (FAB + panel)

styles/main.css          — shared site styles (gallery, nav, header, footer)
styles/viewer.css        — full-page viewer layout (tabs, PiP, mobile)
styles/staff.css         — staff page layout

scripts/build.js         — scans teams/, writes teams/manifest.json
scripts/new-team.js      — CLI to scaffold a new team folder
amplify.yml              — Amplify build config (LFS pull + build.js)

teams/manifest.json      — auto-generated — do not edit by hand
teams/_template/         — scaffold for new teams (underscore = excluded from manifest)
```

### How the gallery works

1. `index.html` loads `js/gallery.js`
2. `gallery.js` fetches `teams/manifest.json`
3. Each entry in the manifest becomes a clickable **card** showing thumbnail, title, team name, tags, and media badges (Poster / Video)
4. The search box filters cards in real-time across: team name, project title, description, tags, and member names

### How the viewer works

Navigating to `teams/{slug}/` loads `index.html` in that folder, which includes `viewer.js`. The viewer:

1. Fetches `./project.json`
2. Builds a full-page viewer with Poster and Video panels
3. Determines initial mode from `project.json` (`defaultMode` field or auto-detected)
4. Loads Supabase + `qa.js` for the Q&A panel

## Viewer

### Modes

| Mode | Main panel | PiP overlay |
|------|-----------|-------------|
| `poster` (default when both exist) | Poster | Video in PiP |
| `video` | Video | — |
| `swapped` | Video (main) | Poster in PiP |

Set `"defaultMode": "video"` in `project.json` to open in video-first mode.

### Controls

| Control | Action |
|---------|--------|
| **POSTER / VIDEO** tabs | Switch main panel |
| `P` key | Switch to poster |
| `V` key | Switch to video |
| `Space` | Play / pause video |
| `F` key | Toggle fullscreen on video |
| `←` / `→` arrow keys | Cycle between available modes |
| Swipe left / right (touch) | Cycle modes |
| Double-tap video (touch) | Toggle fullscreen |
| Pinch (touch) | Zoom in/out on poster or video |
| Double-tap pinch-end (touch) | Reset zoom |
| `Ctrl + scroll` | Zoom in/out |
| **⇄ swap** button (PiP) | Swap main ↔ PiP content |

| **⤢ maximize** button (PiP) | Expand / collapse PiP |
| Drag PiP dragbar | Reposition PiP |

| Extension | Rendered as | Works in WebView? |
|-----------|-------------|-------------------|
| `.png` / `.jpg` / `.webp` / `.gif` | `<img>` | Yes |
| `.pdf` | `<embed>` | Desktop only — blank in mobile WebViews |

## project.json Reference

```json
{
  "name":        "Team 01",
  "title":       "Project title shown in header and gallery card",
  "members":     ["Student A", "Student B", "Student C"],
  "description": "One-line summary shown on the gallery card.",
  "thumbnail":   "assets/thumbnail.png",
  "poster":      "assets/poster.png",
  "video":       "assets/video.mp4",
  "defaultMode": "poster",
  "tags":        ["generative-ai", "rag", "llm"]
}
```

| Field | Required | Notes |
|-------|----------|-------|
| `name` | Yes | Team label ("Team 01") |
| `title` | Yes | Project title |
| `members` | No | Used in gallery search |
| `description` | No | Short blurb on gallery card |
| `thumbnail` | No | Path relative to team folder. Missing → gradient placeholder |
| `poster` | No | Path to poster image — **use PNG**, not PDF |
| `video` | No | Path to local `.mp4` or YouTube/Drive iframe URL |
| `defaultMode` | No | `"poster"` (default) or `"video"` — sets which panel opens first |
| `tags` | No | Lowercase, hyphen-separated; used for search and filter |

## Adding a Team

### Option A — Script (recommended)

```bash
node scripts/new-team.js team-01 "Team Name"
# Creates teams/team-01/ with project.json + README.md
# Regenerates teams/manifest.json automatically
```

### Option B — Manual

1. Copy `teams/_template/` → `teams/team-01/` (lowercase, hyphens only)
2. Edit `teams/team-01/project.json`
3. Add `assets/poster.png`, `assets/video.mp4`, `assets/thumbnail.png` (tracked by Git LFS)
4. Run `node scripts/build.js` to regenerate `manifest.json`
5. Commit and push — Amplify deploys automatically

### Folder naming rules

- Lowercase letters, numbers, hyphens only
- No spaces, no underscores
- Prefix with `_` to exclude from the manifest (e.g. `_template`, `_draft-team`)

## Asset Guidelines

### Posters

- **Format: PNG** — always convert PDF posters before committing
- PDF posters work on desktop browsers only; they are silently blank inside the Metasteps WebView

```bash
python convert-posters.py   # batch-converts poster.pdf → poster.png using ImageMagick
```

- Recommended: 16:9 or A0/A1 aspect ratio
- Stored in Git LFS (`.gitattributes` tracks `*.png`, `*.pdf`, `*.mp4`)

### Videos

- **Format: MP4** (H.264 recommended for widest compatibility)
- Keep under 200 MB per file — larger files slow LFS pulls
- Alternative: embed a YouTube or Google Drive link as an iframe in `README.md`

### Thumbnails

- Any web image format (PNG, JPG, SVG, WebP)
- 16:9 aspect ratio recommended for clean gallery cards
- Target size: under 500 KB

## Deployment

### AWS Amplify

Pushing to `main` triggers an automatic deploy:

```yaml
# amplify.yml (simplified)
preBuild:
  - curl … git-lfs binary → /tmp/git-lfs-3.6.0
  - export PATH="$PATH:/tmp/git-lfs-3.6.0"
  - git lfs install && git lfs pull   # pull all poster/video assets
  - node scripts/build.js             # regenerate teams/manifest.json
artifacts:
  baseDirectory: /
  files: ['**/*']                     # deploy everything as static files
```

Amplify's default build image has no `git-lfs` binary — `amplify.yml` installs it manually from the GitHub releases tarball before pulling.

### GitHub Actions

`.github/workflows/build-manifest.yml` auto-commits an updated `manifest.json` on every push to `main`, keeping it in sync even for local development where you may not have run `build.js`.

### Deployment summary

| Trigger | Effect |
|---------|--------|
| Push to `main` | Amplify: LFS pull → build.js → deploy |
| Push to `main` | GitHub Actions: run build.js → commit manifest |

## Local Preview

`fetch()` is blocked over `file://` — you must use a local HTTP server:

```bash
# Option 1 — npx (no install needed)
npx serve .
# open http://localhost:3000

# Option 2 — Python
python -m http.server 8000
# open http://localhost:8000
```

After adding a team or editing a `project.json`, regenerate the manifest first:

```bash
node scripts/build.js
```

## Reusing for a Future Cohort

To adapt this gallery for a new semester or course:

1. **Update course info** in `index.html`, `staff.html`, `northeastern.html`, and `feedback.html` (header, footer, semester, professor name)
2. **Reset team folders** — clear `teams/team-01/` through `teams/team-32/` and re-run `node scripts/new-team.js` for new teams
3. **Upload assets** — add `poster.png`, `video.mp4`, `thumbnail.png` via Git LFS for each team
4. **Update Metasteps** — in the Unity/Metasteps project, update each wall's WebView URL to the new deployed team URL
5. **Supabase** — either reuse the same project (questions are scoped per `team_slug`) or create a new Supabase project, run the schema, and update `SUPABASE_URL` + `SUPABASE_KEY` in `js/qa.js`
6. Push to `main` — Amplify auto-deploys
